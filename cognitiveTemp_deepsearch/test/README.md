## 🔍 如何使用验证结果

### 理解"为什么MRR/MAP都是0"

1. **不是计算错误**
   - 计算逻辑完全正确
   - 是参数配置问题

2. **根本原因: 阈值0.6太高**
   ```
   搜索结果相关性分布:
   - 0-0.3 (低相关): ~60%
   - 0.3-0.6 (中相关): ~30%  
   - 0.6+ (高相关): ~10%
   
   阈值0.6要求: 至少60%的结果要≥0.6
   实际情况: 只有~10%的结果符合
   
   所以MRR/MAP接近0是正常的！
   ```

3. **改进方案**
   ```python
   # 选项A: 降低阈值
   threshold = 0.3  # 让~90%的结果被认为"相关"
   
   # 选项B: 动态阈值
   threshold = sorted(relevances)[len(relevances)//2]  # 中位数
   
   # 选项C: 改进相关性算法
   # 用更好的NLP模型计算相关性
   ```

### 改进前后对比

## 📞 常见问题

### Q: 为什么我的指标都是0?
A: 查看搜索结果总数。如果是0，说明该报告没有搜索历史。

### Q: MRR/MAP为0是否表示搜索失败？
A: 不是。通常是阈值设置问题。NDCG是更好的排序质量指标。

### Q: 如何改进MRR/MAP?
A: 两种方法：
1. 降低阈值（快速）
2. 改进相关性计算（根本）

### Q: 可以添加自己的指标吗?
A: 可以。编辑METRICS_EXPECTATIONS字典和验证逻辑。
